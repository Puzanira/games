# Project Roadmap

Планер проекта (INCREMENT_MODEL_SPEC §5): что сделано и что кандидат на
следующий инкремент. Следующий инкремент выбирается на Гейте 1 с
основательницей — кандидаты ниже НЕ предрешены.

## Milestone 1: Playable Vertical Slice

Goal: крошечная играбельная версия, доказывающая ядро («руль ↔ время мира»).

Сделано:

- [x] **Инкремент 1 — `wheel-time-core`** (2026-07-13/14, Гейт 2 пройден,
      см. `docs/increments/2026-07-13-wheel-time-core.md`): один
      грейбокс-экран; колёсико мыши скрабит время мира в обе стороны
      (0.12 с/щелчок — постоянная модель, руль-энкодер тоже будет
      дискретным); два обратимых объекта-функции-времени — платформа
      («выбери фазу») и падающий кирпич («останови мгновение»); перенос
      стоящего персонажа обоими объектами в обе стороны («лифт из отмотки»);
      персонаж на клавиатуре в реальном времени; выход=победа,
      яма=смерть=мгновенный рестарт, R — рестарт. Машинный гейт: EditMode
      20/20 + PlayMode 15/15. **Гипотеза «крутить время руками — весело»
      получила первое живое подтверждение** («это какая-то загадка
      получается — что-то в этом есть»).

- [x] **Инкремент 2 — `collapsing-bridge-rewind-window`** (2026-07-14, Гейт 2
      пройден, см. `docs/increments/2026-07-14-collapsing-bridge-rewind-window.md`):
      **первый НЕобратимый атом (рушащийся мост) + окно отмотки**. Новый экран
      Screen02 (= сцена №0 по Play; Screen01 = №1, покрыт регрессиями).
      Окно реализовано как **плавающий нижний предел таймлайна** (`WorldTimeline`
      получил `HighWater`/`MinTime = max(0, HighWater − окно)`; окно —
      баланс-поле, дефолт 15 с, Screen02 = 6 с), НЕ буквальный буфер снапшотов
      (объекты остаются f(t) — Гейт-1 решение; буфер придёт с путь-зависимым
      объектом). Мост «наводится временем» (секция встаёт к t_a) и рушится
      **неравномерной волной** (умеренная скорость) — необратимость возникает
      из недостижимости точки разрушения за краем окна. Контраст с «лифтом из
      отмотки»: мост при обрушении **роняет** игрока (не несёт) — «ты вне
      времени» (GAME_SPEC экран №3); отмотка собирает мост, но не поднимает
      упавшего. Индикатор окна (`RewindWindowView`): «ОТМОТКА ДО» + метка
      разрушения, краснеет при потере моста навсегда. Машинный гейт: EditMode
      34/34 + PlayMode 23/23. **Дизайн-инсайт Гейта 2 (фикс-луп с откатом):
      скрамбл-проход по неравномерно рушащемуся мосту — это желаемая фича, а
      не эксплойт** (основательница откатила «резкий/непроходимый» вариант).

- [x] **Инкремент 3 — `timed-hazard-dodge`** (2026-07-15, Гейт 2 пройден,
      см. `docs/increments/2026-07-14-timed-hazard-dodge.md`): **первый
      враг-механизм — маятник-лезвие**. Новый экран Screen03 (= сцена №0 по
      Play; Screen02 = №1, Screen01 = №2, оба покрыты регрессиями). Враг —
      смертельная **угловая `f(t)`**: `TimedPendulum` качается вокруг пивота
      углом = амплитуда·sin(2π·t/период) (`PendulumSwingCurve`), полностью
      обратимо колёсиком, **без HingeJoint** (физ-джойнт сломал бы обратимость
      руля). При t=0 лезвие вертикально и перекрывает единственный проход
      (опасно, колёсико обязательно); безопасная фаза = крайнее положение
      качания (просвет ≥ роста). Касание = мгновенная смерть (`ZoneTrigger(Kill)`
      на лезвии-триггере) → чистый рестарт; на лезвие нельзя встать (триггер,
      grounding не видит). `TimedObject` получил **канал поворота** (`RotationAt`/
      `ApplyPoseAt`, дефолт «не вращается» — платформа/кирпич/мост не тронуты).
      Ямы нет: единственная смерть = маятник. Машинный гейт: EditMode 43/43 +
      PlayMode 32/32. **Гейт 2 (фикс-луп): pass 1 нашёл настоящий баг — панель
      победы была невидима (`WinUI` без `RectTransform` ⇒ 0×0), а машинный
      win-тест был ложно-зелёным (проверял `activeSelf`, не видимый размер);
      fix #1 — полноэкранный RectTransform + усиленный тест на видимый `rect` →
      «теперь ок».**

Кандидаты следующего инкремента (решается на Гейте 1, НЕ предрешено):

- **Другие атомы словаря головоломок**: пресс/ловушка с циклом, конфликт фаз
  (напр. два врага в противофазе), второй тип врага/угрозы. (Маятник-враг и
  цикличная платформа уже сделаны.)
- **Обучающие экраны 1–3 из GAME_SPEC** («руль двигает мир» / «останови
  мгновение» / «время можно вернуть») — цепочка обучающих экранов.
- **Система переходов между экранами** (цепочка Screen01→Screen02→Screen03→… ) —
  осознанно отложена в инкрементах 2–3, тело Milestone 2.

## Milestone 2: Core Loop

Goal: цепочка экранов-головоломок из словаря атомов; смерть/рестарт и
переходы между экранами как устойчивый цикл; финальная сцена (ждёт
WORLD.md).

## Milestone 3: Polish + железо

Goal: feel, feedback, UI, аудио, баланс; подключение физического
руля-энкодера (HID/serial — нужен ранний железный прототип, ⏳ в GAME_SPEC)
и работа на автомате.
